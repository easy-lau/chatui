# ChatUI v1.5.0

## Highlights

- Added interactive web-page previews for complete HTML documents returned in chat responses. Each detected page has an independent eye-icon preview action and a download action, including responses that contain multiple pages.
- Web previews load on the first click, preserve scripts, forms, and linked resources in the preview frame, and use a borderless presentation. Browser asset cache versions were updated so deployed clients receive the fix immediately.
- Simplified thinking mode for OpenAI GPT-5 models. The control now uses the native `reasoning_effort` values: `low`, `medium`, `high`, `xhigh`, and `max`; disabled thinking sends no reasoning parameter.
- Removed obsolete multi-provider thinking compatibility, token-budget mapping, silent reasoning downgrade retries, and the unused Managed access token configuration.

## Engineering and reliability

- Simplified server runtime wiring and removed redundant compatibility modules, checks, and ownership/auth scaffolding that were no longer part of the product flow.
- Kept release validation in the tag workflow: release metadata is verified before the full project check runs, then amd64/arm64 Docker images and the GitHub Release are published.
- Updated architecture, development, README, and repository guidance to match the streamlined runtime and GPT-5 reasoning behavior.
- Added regression coverage for web-preview detection, multi-page actions, first-open behavior, borderless iframe styling, and reasoning compatibility cleanup.

## Upgrade notes

- No data migration is required.
- Existing Managed access token values are no longer used and may be removed from deployment configuration.
- If a saved session contains a legacy reasoning preference, choose one of the supported GPT-5 effort values or turn thinking off before sending a new request.
- Deployments should refresh browser assets after upgrading; the new release cache versions handle normal refreshes automatically.
