# ChatUI v1.5.6

## Image session switching

- Fixed completed generated images briefly returning to the “generating/loading” state after switching conversations.
- Reuses the already available browser object URL instead of hiding the image and rereading it from IndexedDB.
- Restores multiple persisted media elements concurrently when a database read is required.
- Releases cached object URLs when their stored images are deleted or the image database is cleared.

## Browser delivery

- Bumped the static bundle cache version to `1.3.106-image-session-switch`.
- Bumped the media workflow module cache version to `1.2.68-image-session-switch`.

## Verification

- Added regression coverage for immediate session switching after image generation.
- Added regression coverage ensuring a visible completed Blob image is not returned to a loading state.
- Docker deployments can upgrade to the `1.5.6` or `v1.5.6` image tag.
