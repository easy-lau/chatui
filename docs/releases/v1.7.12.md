# ChatUI v1.7.12

Released on July 21, 2026.

## Fixes

- Fixed completed chat responses leaving the composer permanently stuck in the running/stop state when an IndexedDB session snapshot write stalls.
- Added bounded IndexedDB open, read, write, delete, and clear transactions with unhealthy-connection reset and late-connection cleanup.
- Added self-healing per-session snapshot queues with newest-revision preservation, transient-error retries, exponential backoff, and deletion-safe cancellation.
- Added recoverable localStorage snapshot fallbacks that preserve canonical history without misreporting local data as a durable IndexedDB commit.
- Added quota-aware full, incremental, and minimal fallback formats, plus deterministic merging with the latest durable snapshot.
- Added immediate fallback persistence for browsers where IndexedDB is unavailable and bounded snapshot flushing during page leave.
- Refreshed browser and static-bundle cache versions so deployed clients receive the persistence fix immediately.

## Verification

- Added regression coverage for stalled database opens, reads, and writes; connection rebuilding; retry recovery; newest-revision coalescing; retry cancellation; bounded flushing; permanent storage errors; quota fallback compaction; durable/fallback merging; and browsers without IndexedDB.
- Verified the failure mode in a real Microsoft Edge session with an injected stalled final snapshot transaction: the completed answer remained recoverable and the composer returned from `停止` to `发送` after the bounded commit wait while IndexedDB recovery continued in the background.
- `npm run check` passes with 386 automated tests.
