# ChatUI v1.6.10

## Restored conversations render reliably

- Fixes a startup warning that appeared when reopening a conversation containing completed user and assistant messages.
- Prevents completed persisted conversations without an active task record from dereferencing a missing task state during session recovery.
- Keeps normal history rendering, task cleanup, composer availability, and session busy indicators synchronized after a refresh.
- Updates browser bundle cache versions so deployments immediately load the corrected task lifecycle code.
