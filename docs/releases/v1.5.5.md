# ChatUI v1.5.5

## Current snapshot format only

- Removed legacy per-session local-storage history migration.
- Removed IndexedDB snapshot v1 migration and partial-history reconciliation against legacy backups.
- Session startup and live snapshot reload now accept only current snapshots with `snapshotVersion >= 2` and a valid message list.
- Removed the legacy display-to-message migration API and its recovery-specific helpers.

## Behavior change

- Legacy local-storage histories and v1 IndexedDB snapshots are intentionally ignored and are no longer converted or written back.
- Existing v2 snapshots continue to load normally, including pending display items and the latest generated image reference.

## Browser delivery

- Bumped the static bundle cache version to `1.3.105-current-snapshot-only`.
- Bumped the session display module cache version to `2.1.6-current-snapshot-only`.

## Verification

- Replaced compatibility migration tests with current-format-only loading tests.
- Covered v2 loading, ignored legacy local-storage history, ignored v1 snapshots, and live-reload version enforcement.
- Docker deployments can upgrade to the `1.5.5` or `v1.5.5` image tag.
