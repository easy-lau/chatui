# ChatUI v1.6.8

## Immediate message projection hotfix

- Fixes newly sent messages being persisted successfully but not appearing immediately in the current page until refresh.
- Assigns canonical order indexes to user messages in chat, image generation, and image editing modes instead of leaving image-mode indexes blank.
- Writes the canonical message state before rendering the DOM projection, preventing render and virtualization code from observing an incomplete submission.
- Uses one guarded display-order insertion method so blank or invalid indexes can no longer move a fresh message to the start of history or outside the visible tail.

## Verification

- Added regression coverage for blank display indexes, cross-mode canonical message indexing, and canonical-before-DOM submission ordering.
- Verified consecutive image-mode submissions in a real browser; the second message appears immediately at the conversation tail without refresh.
- Verified the complete project check suite with 328 passing tests.
- Docker deployments can upgrade to the `1.6.8` or `v1.6.8` image tag.
