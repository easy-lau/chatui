# ChatUI v1.7.2

## Highlights

- Treats multi-image composition as resource selection plus image upload and sends the user's composition request to the image API unchanged.
- Locates named historical images semantically, uploads only the selected images through repeated `image[]` fields, and asks for clarification only when identity is genuinely ambiguous.
- Fixes the intent-recognition model's "follow chat model" setting so clearing an old explicit route model is persisted instead of silently restoring it.
- Makes intent recognition follow the chat model selected for the target session, including background sessions, session switches, route review, and explicit-route fallback.
- Records the model actually used in the in-memory intent trace to make routing configuration verifiable without persisting sensitive request data.

## Validation

- Full project checks and 361 automated tests.
- End-to-end image smoke test verifies semantic resource selection, exact prompt passthrough, repeated `image[]` multipart fields, and exclusion of unrelated images.
- Route payload tests verify two sessions use their own selected chat models, an explicit route model still takes precedence, and fallback uses the session model instead of the global default.
- Headless Microsoft Edge verification against the built static bundle covers exact image prompt passthrough and session-aware intent-model payload selection with no page errors.
